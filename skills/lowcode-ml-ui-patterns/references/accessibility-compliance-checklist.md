# Accessibility Notes For Low-Code ML UIs

- Provide keyboard add, select, move, connect, disconnect, delete, and configure operations for canvas nodes.
- Give nodes, ports, and edges accessible names and visible focus states.
- Announce execution state changes without flooding live regions.
- Provide a textual workflow summary or outline for screen reader and audit paths.
- Invalid connections should explain incompatible types or missing prerequisites.
- Do not rely on color alone for chart or node status; pair color with labels, shape, icon, pattern, or text.
- Provide table equivalents or summaries for critical charts.
- Keep drag-and-drop operations achievable without dragging.
