# Training Curriculum Deck

Use this skill for decks meant to teach, train, onboard, or facilitate practice. Training specializes a concept or a small set of concepts deeply; technical playbooks cover broader systems and minute implementation details for self-study.

Read `../../references/training-deck-design.md` for substantial work. Read `../../references/powerpoint-production-handoff.md` before creating a PPTX.

## Workflow

1. Define learner profile, prerequisites, environment, timebox, and measurable success outcome.
2. Select the concept scope: one concept deeply, or a small cluster of connected concepts.
3. Plan each segment as explain, show, practice, feedback, check, and transfer.
4. Lock a learning visual system: domain theme, mode markers, icon family, example style, practice/check color coding, and accessibility rules.
5. Use visuals that teach: annotated screenshots, diagrams, worked examples, contrast pairs, source figures, flow charts, videos/stills, or realistic domain images.
6. If a concept has standard diagrams or research visuals, use permitted/user-provided assets directly or recreate them as editable teaching diagrams with citation.
7. Add facilitator notes, timing, prompts, answer keys, and debriefs when useful.
8. If producing `.pptx`, invoke the `Presentations` skill once after this strategy is set.

## Blocking Gates

- No lecture-only bullet deck.
- No generic theme when the training has a domain.
- No activity without instructions, timebox, success criteria, and answer/debrief.
- No decorative imagery that does not teach.
- No unexplained source figure, screenshot, or chart.
