# Training Deck Design Reference

Keep this reference lean. Do not bulk-load external source pages because a source URL exists; browse only topic-specific sources needed for evidence, images, diagrams, examples, or citations.

## Training Deck Purpose

Training specializes one concept or a small concept cluster deeply. It should help learners practice, get feedback, and transfer the concept to real work. If the goal is broad technical coverage with lots of implementation detail, use the technical playbook plugin instead.

## Required Design Brief

Before creating slides, define:

- Learner: role, prerequisites, context, anxiety points, and target behavior.
- Scope: concept or small concept cluster.
- Outcome: measurable behavior, not just "understand".
- Learning loop: explain, show, practice, feedback, check, transfer.
- Visual system: domain theme, mode markers, icon family, example style, practice/check colors, accessibility rules.
- Asset plan: screenshots, diagrams, source figures, videos/stills, realistic examples, generated or recreated visuals.

## Learning Slide Types

- Opener: outcome, relevance, time, prerequisites.
- Concept: one idea, visual explanation, short narration.
- Worked example: step-by-step reveal, annotated artifact, why each step matters.
- Contrast pair: wrong/right, before/after, novice/expert, risk/safe.
- Practice: task, materials, timer, success criteria.
- Knowledge check: question, options, feedback, explanation.
- Scenario: context, choice, consequence, debrief.
- Transfer: how to apply it in the learner's real domain.

## Visuals That Teach

- Use annotated screenshots, flow charts, decision trees, maps, timelines, examples, and source figures when they reduce abstraction.
- Use permitted/user-provided/open visuals directly; recreate unclear-source visuals as editable diagrams and cite conceptual sources.
- Use icons as recurring learning-mode markers, not decoration.
- Use domain imagery when it grounds the lesson: finance statements, mine/process photos, software UI, clinical pathways, industrial systems, etc.

## QA Gates

- Outcomes are measurable.
- Each segment includes explain, show, practice, feedback, and check.
- Visuals carry instructional meaning.
- Activities include instructions, timebox, success criteria, and feedback.
- The deck has a domain-specific or learner-appropriate theme.
- Important visuals have citation/attribution or were recreated as original diagrams.
- Accessibility: readable type, sufficient contrast, captions/summaries for media, no color-only encoding.
