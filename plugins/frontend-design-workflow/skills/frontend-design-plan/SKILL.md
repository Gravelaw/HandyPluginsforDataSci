# Frontend Design Plan

Use this skill to turn an interface idea into design intent, constraints, and execution criteria.

## Workflow

1. Restate the product goal, primary task, audience segments, and platform constraints.
2. Identify the likely user modes: first-run, routine use, expert use, recovery from error, and collaboration or sharing.
3. Use the six-layer framework in `../../references/design-framework-layers.md` to select only the relevant theory, strategy, compliance, architecture, domain, and engineering concerns.
4. Define the information architecture: navigation, core objects, primary workspace, secondary panels, search, settings, and empty states.
5. Specify interaction principles: progressive disclosure, feedback loops, undo/recovery, keyboard paths, loading states, and error localisation.
6. Choose visual direction appropriate to the product category. Operational tools should be dense, restrained, and scannable; consumer products can be more expressive.
7. Define acceptance criteria before implementation: accessibility target, responsive breakpoints, data states, visual QA, performance budget, and test surface.

## Output Shape

Prefer a concise plan with:

- Goal
- Personas or user modes
- Information architecture
- Core flows
- Design principles
- Implementation implications
- Validation and risks
